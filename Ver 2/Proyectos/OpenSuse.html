# Instalacion

Se necesitan como requerimientos minimos: Procesador de 64-bits con 2GHz, 2GB RAM y 4GB de Disco Duro. Se recomienda tener un USB o DVD y una coneccion estable a Internet.

“«La Respuesta a la Gran Pregunta de la vida, del universo y de todo es... Cuarenta y dos. » -dijo Pensamiento Profundo con infinita majestuosidad y calma."- Douglas Adams (The Hitchhiker's Guide to the Galaxy 1979)

Pasos para iniciar el instalador en Windows:

  1. Realizar un Backup de tu Informacion.
  2. Desfragementar el disco duro y realizar operaciones de limpieza.
  3. Quemar el ISO de la distribucion en un USB con ImageUSB.
  4. Si ImageUSB no funciona utilizar Rufus. GPT for UEFI, FAT32, DD Option.
  5. En linux seria dd if=(iso) of=(USB) bs=4M
  6. Acceder al Boot apretando la tecla F9 al iniciar el sistema.
  7. Desactivar las opciones de Secure Boot y Legacy Boot. Poner al USB como primer dispositivo Boot.
  8. Reiniciar con el USB insertado, iniciando desde el USB con el instalador de OpenSUSE.

Los ultimos sucesos relacionados con openSUSE son publicadas en el portal de Noticias y en sus redes sociales de YouTube y Twitter.

El sitio principal cuenta con un Buscador de Software, un Feed de Articulos, un Foro General Si existe alguna duda, tambien esta disponible la Documentacion Oficial, las Guias Wiki y la Guia General.

# Instalador

Una vez que se inicio el instalador YaST podemos comenzar con los pasos para instalar el sistema operativo. Al inicio se nos presentan una serie de opciones principales y una serie de opciones secundarias (F1-F9).

Opciones Principales:

  1. Live Boot: Boot openSUSE desde el dispositivo, para realizar un preview.
  2. Install: Instalacion de openSUSE. Cambiar opciones F5 si el kernel no es compatible.
  3. Upgrade: Actualiza los componentes en el caso de que ya este instalado openSUSE.
  4. Rescue: Interfaz minima necesaria para realizar cambios en el sistema operativo.
  5. Check Media: Revisa la integridad delos componentes, realizar antes de una nueva instalacion.
  6. Memory Test: Prueba de RAM para ver si no hace falta memoria o existe memoria corrupta.

Opciones Secundarias:

  1. Ayuda: Muestra informacion adicional de las opciones basicas.
  2. Lenguaje: Lenguaje de la interfaz del instalador.
  3. Modo de Video: Al iniciar generalmente KMS detecta la resolucion, si la resolucion es incorrecta esta opcion nos permite elegirla manualmente. Seleccionamos (No-KMS) y (vga=ask). En casos exepcionales podemos desactivar la interfaz grafica del instalador con la opcion (vga=no), esto activa el modo de texto.
  4. Dispositivo Desde aquí se selecciona el medio de instalacion ya sea DVD, USB, FTP, NFS, SLP etc.
  5. Kernel: Con esta opcion podemos configurar algunas opciones del Kernel que pueden ser la causa de errores en la instalacion. Por ejemplo ACPI podemos desactivarlo con (NO ACPI). Tambien podemos activar la opcion (Safe Settings) que realiza la instalacion en modo DMA (Solucionando problemas con la lectura de CD/DVD) y desactiva las funciones del Power Managment. Por ultimo se puede intentar desactivar el APIC de la computadora con la opcion (NO APIC).
  6. Drivers: Permite descargar drivers adicionales proporcionando un URL para que descarge los drivers y los integre al proceso de instalacion.
  7. Arch: Permite seleccionar la arquitectura del sistema operativo. Las ultimas versiones unicamente soportan arquitecturas de 64bit.

Al iniciar el instalador se sigue una serie de pasos que nos permite modificar el estado inicial del nuevo sistema:

  1. Lenguaje y Contrato: Selecciona el lenguaje del sistema operativo y muestra licencia GNU.
  2. Configurar Red: Configura opcionalmente la red de internet para instalar los ultimos paquetes de seguridad.
  3. Repositorios: Si se configuro una red de internet en el paso anterior, aqui se podran seleccionar opcionalmente los repositorios que agregar por defecto.
    • Main OSS: Packetes fundamentales y software Open Source del sistema Operativo.
    • Main NON OSS: Paquetes propietarios adicionales.
    • Update OSS: Contiene las ultimas actualizaciones de los paquetes OSS.
    • Update NON OSS: Contiene las ultimas actualizaciones de los paquetes NON OSS.
    • Main Sources: Contiene el codigo fuente y Spec Files de los paquetes OSS.
    • Main DEBUG: Contiene paquetes opcionales para probar y corregir el sistema.
    • Update DEBUG: Actualiza el repositorio DEBUG.
    • Untested: Paquetes experimentales que estan siendo probados por los beta testers.
  4. Particion: Selecciona la particion donde se desea usar para instalar el sistema operativo. OpenSUSE, como muchos sistemas operativos Linux, utiliza generalmente cuatro subparticiones: Particion de Montaje EFI (/boot/efi), SWAP (1-2GB), Home BtrFS (30 - ? GB) y Root Ext4 (20 - 100GB). El bootloader GRUB sobreescribe automaticamente el MBR.
  5. Zona horaria: Selecciona la hora del sistema operativo. UTF por defecto.
  6. Paquetes: Este paso solo aparece si se selecciono en el tercero la instalacion de algunos paquetes adicionales.
  7. Entorno del Escritorio: En este paso se selecciona el entorno del escritorio a utilizar. GNOME se parece más a otras distribuciones de linux. KDE es preferido por muchos por su similitud en apariencia a Windows y Mac. Si buscas una interfaz minimalista se recomienda LXDE o XFCE. Por ultimo la opcion de MinimalXWindow es la interfaz más simple recomendada unicamente para servidores, utiliza comandos de texto para la mayoria de sus acciones. Si no se desea ningun entorno se puede seleccionar el modo Minimal Server Selection, en este modo solo se dispone de la terminal.
  8. Usuario y Root: En este paso se crea el usuario local con su contraseña. Tambien se establece la contraseña del usuario maestro "Root". Se recomienda ampliamente que la contraseña de usuario y de root sean diferentes.
  9. Review: Muestra todo lo que va a realizar el instalador, permite regresarse y hacer modificaciones de ultimo momento. ALgunas opciones solo se pueden modificar aqui como Software, Booting, Firewall y System Target, el cual es por defecto el entorno de escritorio seleccionado anteriormente.

# YaST

Linux es un sistema multi-usuario. Varios usuarios pueden estar activos a la vez, por lo que se le asigna a cada uno un UID Unico. El UID establece los permisos de cada usuario.

Para administrar a los usuarios de una computadora utilizamos el comando sudo yast2 users &. Dentro de esta aplicacion podemos modificar, eliminar, crear o desactivar cuentas de usuario.

Tambien podemos mapear el ID local con el de la red, activar login automatico, encriptar directorios Home, establecer limites de recursos y expiracion de contraseñas.

Entre los metodos de autenticacion se encuentran NIS, LDAP, Kerberos y Samba. Incluso despues de la instalacion podemos modificar facilmente el idioma y zona horaria del sistema.


#Modificar el lenguaje del sistema
sudo yast2 language &
#Una vez instalados los idiomas iniciar una aplicacion con otro idioma como:
LANG=de_DE Chrome
#Para obtener los codigos de los idiomas utilizamos:
locale -av
#Para dual boot a veces es necesario desactivar el reloj del Hardware
sudo yast2 timezone &
#Cambiar configuracion del teclado o del sonido
sudo yast2 keyboard &
sudo yast2 sound &
#Si existen problemas con los drivers de sonido se recomiendan:
zypper in pavucontrol
zypper install alsa-firmware
#Para ver las fuentes instaladas en el sistema
fc-list ':' family
#En donde fc-list es un wrapper de Fontconfig
#Para activar el visor de fuentes utilizamos el sig comando
ftview 14 /usr/share/fonts/truetype/FreeMono.ttf
#Ver las particiones disponibles
/sbin /fdisk -l
#Montar la particion NTFS
ntfs-3g /dev/(nombre particion)
#Montar en solo lectura y desmontar
ntfs-3g /dev/(nombre particion) -o -ro
fusermount -u (nombre particion)
#Para montar archivos iso
fuseiso (Archivo) (Particion)
#Para checar las actualizaciones del sistema
sudo yast2 online update

Al utilizar openSUSE se pueden presentar una serie de problemas comunes. Para encontrar informacion sobre un problema del sistema operativo se recopila informacion en archivos de texto, localizados en /Var/log/ y en /Proc/
