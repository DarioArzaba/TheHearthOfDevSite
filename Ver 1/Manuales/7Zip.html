7zip

Software Open Source (GNU LGPL) para crear archivos comprimidos. Desarrollado por Igor Pavlov. Genera archivos en formato “7z Format” mediante algoritmos de compresión LZMA y LZMA2.

  Caracteristicas

  • Algoritmos de Compresión: Deflate (LZ77) en su forma de LZMA en conjunto con LZMA2, PPMD basado en el PPMdH de Dmitry Shkarin, BCJ - BCJ2 para ejecutables y BZip2.

  • Seguridad: Encriptación AES-256 para 7z.

  • Velocidad: Radio de compresión 30-70 veces el de “Zip Format”.La velocidad de compresión es de aproximadamente 1MB/s en un CPU de 2GHz. La velocidad de descompresión es de 10 a 20 MB/s en un CPU de 2GHz. El código meta de descompresión es de alrededor de 5KB.

  • Revision: El archivador puede agregar comentarios, los cuales se guardan en un archivo de texto llamado “descript.ion”.

  • Formato: 7z es un formato de arquitectura modular abierta, es decir que es compatible con compresión, conversión o encriptación. El formato soporta archivos de 16 ExbiBytes y compresión solida o por partes.

  • Tamaño: Los archivos comprimidos con nuevas versiones (+15.6) son más grandes debido a que se ha sustituido el sistema de ordenamiento “Por Extensión”, ahora se utiliza el sistema “Por Nombre”. Al agregar (qs) o (–mqs) a la consola podemos activar el sistema “Por Tipo de Archivo”, reduciendo el tamaño, notece que este ordenamiento no es soportado por NTFS el cual utiliza el sistema “Por Nombre” por defecto. Si se busca reducir el tamaño del archivo se recomienda aumentar el tamaño del diccionario.

  • Errores: Si 7Zip no puede descomprimir un archivo en formato Zip, generalmente es por tener “Headers” incorrectos, estos son insertados cuando la compresion se realizo en un programa no compatible con 7Zip.

  • Compatibilidad: Puedes agregar la funcionalidad de 7zip a una aplicacion independiente incluyendo la libreria (7za.dll), de LZMA SDK. Esta libreria utiliza Interfaces COM. Con LZMA SDK incluso puedes diseñar tu propia libreria de compresión.

  LZMA SDK

El LZMA Software Development Kit nos permite desarrollar librerias de compresion basadas en LZMA. Este Kit contiene:

  • C++ Source Code de LZMA (Codificador y Decodificador) y la libreria para utilizar 7z.

  • ANSI-C Source de LZMA / LZMA2 / XZ (Compresion y Descompresion) y librerias para descomprimir 7z.

  • C# Source de LZMA (Compresion y Descompresion).

  • Java Source de LZMA (Compresion y Descompresion).

  • lzma.exe para el formato .lzma (Compresion y Descompresion).

  • 7zr.exe version minificada de 7zip por si se quiere añadir a una aplicación.

  • Modulos SFX para la creacion de paquetes auto-extraibles.

Compresión

Comprimir: Las partes repetitivas de un archivo son sustituidas por variables disminuyendo su tamaño. Las variables se guardan en un diccionario.

Archivo Comprimido Dentro de un archivo comprimido se encuentran una serie de variables que representan a la información original.

Descomprimir: Volver a sustituir las variables del diccionario por el contenido original.

  Compresíon Lossless

La compresión se denomina "Lossless" debido a que no se pierde información. El metodo (por diccionario) se originó en el sistema “LZ Adaptative Dictionary Based Algorithm”, en donde LZ se refiere a Lempel y Ziv, sus desarrolladores. El algoritmo es Adaptativo porque el programa analiza la información en busca del mejor patrón de compresión. Cada vez que encuentra un nuevo patrón lo guarda en el diccionario. Cuando no encuentra más patrones comparara los ya guardados y utiliza el que generó la mayor compresión.

  Compresíon Lossy

En esta compresión no se sustituye la informacion repetitiva sino que se elimina la innecesaria. El archivo es permanentemente recortado. Este metodo no funciona con archivos de texto pues el programa no sabe distinguir que informacion es necesaria, sin embargo en archivos como imagenes o audio se pueden eliminar ciertas partes promedio comprometiendo la calidad (Pixeles o Bits/Audio) pero manteniendo la coherencia general.
